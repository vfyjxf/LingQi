<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User-controlled PR data -> AI prompt | PR title/body/files flow into the prompt string, making prompt injection possible |
| AI prompt -> AI model | The prompt text is the sole instruction channel for scoring behavior |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-05 | Tampering | buildReviewPrompt() — PR title/body in prompt | accept | User-controlled PR metadata could bias scores, but the scoring section operates on diff content, not title. Structural separation in prompt reduces this; residual risk accepted for v1. |
| T-06-06 | Denial of Service | AI output token limit | mitigate | maxOutputTokens increased to 8000 in Plan 01; if still insufficient, config allows up to 16000 |
| T-06-07 | Spoofing | Model produces reasoning text instead of JSON | mitigate | Anti-reasoning instruction in both system prompt and user prompt: "不要输出推理过程或自然语言说明，只输出 JSON 对象"; if parse fails, Zod validation surfaces clear error |
| T-06-08 | Information Disclosure | Evidence field | accept | evidence field may quote sensitive code snippets; this is the intended behavior (DIM-04 requires evidence anchoring). Review output is not persisted or transmitted to third parties. |
</threat_model>
